## 搬瓦工 VPS 更换机房的完整步骤

以下是完整操作流程，按顺序来。

**1. 登录搬瓦工账号后台**

打开搬瓦工官网（国内一般用 bwh81.net），登录账号。进入 Services → My Services，找到你的 VPS，点右边的 "KiwiVM Control Panel" 按钮。

**2. 进入 KiwiVM 面板**

这是搬瓦工自己开发的控制面板，机房切换、重装系统、查流量这些操作都在这里。

**3. 找到 "Migrate to another DC"**

左侧菜单往下翻，找到这个选项。点进去之后会显示你当前所在机房，以及可以迁移的机房列表。

**4. 选择目标机房**

从列表里选你想去的机房，然后点 "Confirm on next step"。

不同套餐可选的机房不同，CN2 GIA-E 套餐可以看到的机房会比 KVM 套餐多很多。

**5. 确认并开始迁移**

进入下一步之后，点 "Start Migration" 正式开始。

迁移时间取决于你的 VPS 已用磁盘量。空间用得少的话，几分钟内就完成了；数据多的情况下可能需要二三十分钟甚至更长。

**6. 等邮件通知，更新 IP**

迁移完成后，搬瓦工会发邮件通知你。这时候 VPS 的 IP 地址已经变了——如果你有域名解析到这台 VPS，记得回到 DNS 控制台更新 A 记录。

## 换机房可以换几次？有次数限制吗？

没有硬性次数限制，也不收费。

但每次迁移完成之后有个冷却期，不能立刻又切。具体冷却时间以 KiwiVM 面板显示为准。如果迁移完发现不满意，等冷却期过了可以再换回去或者换别的机房。

有一点要注意：KVM 套餐里有几个机房（比如 DC8 CN2 和 DC3 CN2）迁过去之后流量会变为原来的三分之一，这是官方规则。面板上切换时会有提示，看清楚再确认。

## 换机房之后 IP 变了，网站怎么办？

这是换机房之后最常见的后续问题。

IP 一定会变，这是迁移的机制决定的。处理步骤如下：

1. 迁移完成后，去 KiwiVM 面板首页查新 IP 地址
2. 登录域名注册商的 DNS 管理后台
3. 把 A 记录从旧 IP 改成新 IP
4. 等 DNS 生效（通常几分钟到一小时，取决于 TTL 设置）

如果你在换机房前把 TTL 改小（比如 300 秒），DNS 更新就会快很多。这个是提前操作省事的小技巧，不提前改也问题不大，就是等的时间可能久一点。
